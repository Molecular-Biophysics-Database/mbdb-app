MBDB - Testing MST deposition

November 2023 - January 2024

What’s the task?

Make a test deposition of a MST dataset and report back the shortcomings or bugs related to:

  1. Missing fields (i.e. I have information abut the measurement I want to give, but I can’t)

  2. Unnecessary fields (i.e I don’t understand why this field is here)

  3. Unclear help text (i.e. I don’t understand what I’m supposed to put in this field)

  4. Layout/functionality of deposition form (i.e. the current layout annoys me for reasons x, y, and z)

Please have a look at the section before reporting, and report back to mbdb@ibt.cas.cz. Other suggestions are of course also welcome, however we’ll prioritise the ones related to above points, as changes to these things will become difficult when the MBDB public.

Why should I do it?

As much as we enjoy writing code in peace and quiet, we’re want MBDB to be useful to people. Towards this end, we would appreciate your input while it’s still not too painful to make changes. As soon as real data come in and people get used doing depositions in a certain way, we’ll be much more restricted in what we can change. Asking for changes now will save yourself and your lab colleagues from future annoyance, and make MBDB a much more useful resource!

Where and how?

You will need an ORCID account to be make a deposition, so please make one here if you don’t have one. The first time you login to the MBDB you’ll be asked for an email address, feel free to give us a fake one as we currently won’t use it.

  1. Please go to the MBDB test server

  2. Proceed by:

    1. Logging in directly by clicking the ORCID logo

    2. Click on deposit (and be redirected to the login page)

  3. Use the form to make a deposition (it isn’t recommended to use a mobile device for this).

A few notes on the resulting record:

  • You can save your work on a deposition at any time by pressing the save button, and come back to it later.

  • Don’t worry about getting all the details perfectly right, your deposition won’t be converted into a long-lived record .

  • Your deposition will be deleted at the end of the testing phase. We’ll of course analyse it, but it will eventually be lost when we launch a new version of the MBDB

  • We currently don’t check that the uploaded measurement file actually contains measurements, so if you don’t have an MST file at hand, feel free to upload a small pdf, xlxs, or similar.

Known shortcomings

  • Individual user rights are not implemented yet, i.e. anybody can change anybody else’s record (please be nice to each other). We’re working on it.

  • Only a small subset of organism, affiliations, and grants are available in this test. Searching the online resources is under development.

  • Metadata is currently not extracted from the measurement files, we’re working on it.

  • Collections of records (Projects and Measurement Groups) aren’t implemented yet.

  • The documentation is still very sparse, we’re working on it.

  • Searching in a more precise way than a text based search is cumbersome at the moment. We’re working on developing a tool to make this easier.

  • DOIs are not currently generated as we would then be required to keep the records indefinitely.

  • Inside the simplified view of a record, pressing "download" next to the files that has been uploaded currently doesn’t give you the actual file, just information about it. We’re working on fixing this.

Extras information

  • If you want to see the full record (in JSON format) you can display it using the "Download entry" button, and save the content to a file if you want.

  • If you want to see the listing of all the records currently deposited, press the search icon with an empty search field.

  • Pressing the title of a listing result give you a simplified you of the record.
